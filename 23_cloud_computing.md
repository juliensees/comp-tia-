Cloud Computing - any case where computer resources over delivered to a customer at a remote location over           a network
      - the resources of the cloud are accessible with the internet form anywhere
      - on-demand network - you can create/destroy cloud resources on n as needed basis 

- Cloud Computing Roles:  
    * Cloud Service Provider - the business that offers the computing services  
        - responsible for building/maintaining their service offerings  
        - can outsource portions of their infrastructure to other CSPs (and are then Cloud Customers                             also)  
    * Cloud Customer - purchase cloud computing services from one or more providers to help run business  
        - they could do everything themselves or have dedicated account reps  
    * Cloud Service Partners - 3rd party companies that offer add-on services  
    * Cloud Access Security Broker (CASB) - a security solution that sits between users and cloud                         services to monitor, control, and secure cloud usage  
        - address shadow IT - unauthorized cloud app usage   
        - basically it's cloud security control  

Drivers for Cloud Computing 
  One-Demand Self-Service - things that would take a company months can happen in seconds because cloud services can automatically increase their capacity for clients (they already have the servers, hardware)

  *Horizontal Scaling - adding more servers to the pool (to meet increased demand)  
  *Vertical Scaling - adds more resources (CPU or memory) to existing servers to meet demand   
  
  ### Cloud Offers Rapid Elasticity - expanding and contracting quickly (servers/hardware/etc)

- Measured Service - clients pay only for what you consume
  - cloud providers measure the number of seconds you use a virtual server, disc space, function calls,                   etc

*Multitenant Computing - multi-tenancy is the concept that different customers share use of the same                   computing resources  
    - allows cloud providers to oversubscribe their resources (balancing different workloads at different           times)
    - Resource pooling - the memory and CPU are shared among many different users 

Isolation - actions of one customer should never affect another customer (principle of security)

* Cloud Considerations - clients make cost-benefits analysis for what makes sense for their business 
      - on-premise costs vs cloud costs
          - Intangible Benefits of Cloud
              - availability, ease of deployment, responsiveness, scalability, resilience, etc

** Security Service Providers

Managed Service Providers (MSP's) - companies that perform tasks that an org considers a commodity or             that can be done more efficiently/effectively than the org themselves

- Managed Security Service Provider (MSSP's) - can provide a range of security services to an org
    - can provide an entire security infrastructure, monitor system logs, manage firewalls, etc
  * Security as a Service (SECaaS) - outsourcing security function to a cloud provider
        - 3rd party risk (another company is now in charge of your own security)
        - data privacy - your data now passes through their system
        - internet dependency - connection goes down, security may be affected
        - vendor lock-in - switching providers might be diffficult
    Some possibilities:
      - IDaaS (Identity as a Service) - cloud based identity/authentication management
      - CASB (Cloud Access Security Broker) - security for cloud app usage & shadow IT
